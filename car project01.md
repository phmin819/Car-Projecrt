# 3D 주차 시뮬레이터 프로젝트 결과 보고서

본 문서는 Android WebView 환경 기반의 3D 주차 시뮬레이터 프로젝트(Car Project) 개발 과정에서 적용된 핵심 기술 및 구현 사항을 요약한 문서입니다.

## 1. 프로젝트 개요
순수 JavaScript와 Three.js 라이브러리를 활용하여 웹 기반 3D 환경을 구축하고, 이를 Android 애플리케이션의 WebView와 연동하여 동작하는 고해상도 주차 시뮬레이터를 개발하였습니다. 단순한 아케이드 방식이 아닌 실제 차량의 거동을 모사하는 것을 목표로 하였습니다.

## 2. 핵심 구현 기술 및 알고리즘

### 2.1. 애커먼 조향 구조(Ackermann Steering) 기반 물리 엔진
- 단순 제자리 회전을 배제하고, 실제 차량의 앞바퀴 조향각(Steering Angle)에 따른 곡선 주행 궤적을 수학적으로 구현하였습니다.
- 삼각함수(sin, cos)를 활용하여 차량의 현재 헤딩(Heading) 각도와 조향각을 결합, 전진 및 후진 시 차량의 중심축이 완벽한 원호(Arc)를 그리며 이동하도록 벡터 연산을 최적화하였습니다.

### 2.2. 매트릭스 변환을 활용한 체이스 카메라(Chase Camera)
- 차량의 방향에 관계없이 시점이 요동치는 현상(Gimbal Lock 방지)을 해결하기 위해, 차량 객체의 로컬 좌표계(`matrixWorld`)를 기준으로 카메라의 위치를 계산하였습니다.
- `applyMatrix4` 메서드를 통해 차량 후방의 고정된 오프셋 좌표를 절대 좌표로 변환하고, `lerp`(선형 보간) 함수를 적용하여 시점이 차량을 부드럽게 추적하는 레이싱 게임 표준 카메라 시스템을 구축하였습니다.

### 2.3. 물리 기반 렌더링(PBR) 및 절차적 모델링(Procedural Modeling)
- 외부 3D 에셋의 로딩 지연을 막기 위해 Three.js의 기본 기하학(Geometry) 객체를 조합하여 차량의 세부 파츠(본네트, 스포일러, 라디에이터 그릴 등)를 절차적으로 생성하였습니다.
- `MeshPhysicalMaterial`을 활용하여 실제 자동차 도면의 질감을 묘사하는 클리어코트(Clearcoat) 광택 효과와 유리창의 반투명 틴팅 효과를 적용해 시각적 품질을 극대화하였습니다.

### 2.4. 캔버스 2D API 기반 실시간 미니맵(Radar) 알고리즘
- 3D 월드의 공간적 정보를 2D 평면으로 사상(Mapping)하는 실시간 네비게이션 시스템을 구축하였습니다.
- HTML5 Canvas를 오버레이로 띄우고, 맵의 크기 비율에 맞춰 3D 공간상의 장애물(AABB), 목표 주차 구역, 그리고 사용자의 차량 위치 및 회전 각도를 2D 캔버스에 프레임 단위로 동기화 렌더링합니다.

### 2.5. 난이도별 다이내믹 환경(Dynamic Environment) 렌더링
- 사용자 난이도(EASY, MEDIUM, DIFFICULT)에 따라 맵의 지형물뿐만 아니라 전역 렌더링 변수들을 동적으로 제어하는 테마 엔진을 개발하였습니다.
- 방향광(Directional Light), 환경광(Ambient Light), 안개(Fog)의 색상과 밀도, 바닥 재질을 제어하여 각 난이도 고유의 환경(숲, 야간 도시, 화산 지대)을 구성하였습니다.
- `requestAnimationFrame` 주기에 맞춰 회전하거나 상승하는 파티클(Particle) 및 점멸하는 조명 객체를 생성하여, 배경이 정적이지 않고 상호작용하는 듯한 애니메이션을 구현하였습니다.

### 2.6. 터치 스와이프 기반 정밀 조향 인터페이스
- 모바일 환경에서의 조작성을 고려하여 가상 스티어링 휠 UI를 구현하였습니다. 화면 터치의 X축 델타(Delta) 값을 정밀하게 수집하여 실제 차량의 최대 조향각에 비례 매핑함으로써 정교한 컨트롤이 가능하도록 최적화하였습니다.
